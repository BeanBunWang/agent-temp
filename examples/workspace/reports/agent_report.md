# Agent 文档处理报告

## 任务
读取 data 目录下的文档并生成报告

## 摘要
本报告由本地 CLI Agent 生成。Agent 先按需加载报告写作 skill，再检索 data 目录、读取源文档、验证 shell 安全边界，并写入报告文件。

## 关键发现
- 推荐采用显式 Agent Loop，而不是把控制权散落在工具或提示词中。
- 工具调用应统一返回结构化结果，失败、拒绝和超时都要进入 trace。
- Skill 只承载流程知识，正文按需进入上下文，避免启动时消耗预算。
- 上下文压缩应保留目标、约束、进度、重要工具结果和未完成事项。

## 来源摘录
# Agent 架构调研笔记

## 背景

这个示例文档用于验证本地 CLI Agent 的多步处理能力。任务要求 Agent 能读取
workspace 内的资料，按需加载 skill，生成一个可复盘的报告，并导出完整 trace。

## 架构要点

1. Agent Loop 应该是显式状态机，而不是隐藏在提示词或工具实现中。
2. 工具调用必须有统一协议。成功、失败、拒绝、超时都应该归一化为 ToolResult。
3. Skill 是流程知识，不是绕过权限的执行通道。启动时只放元数据，正文按需加载。
4. 上下文压缩不能丢失任务目标、关键约束、当前进度、重要工具结果和未完成事项。
5. Trace 应该记录模型调用、工具调用、skill 加载、token 估算、压缩、边界事件和耗时。

## 边界情况

- 文件读取需要限制在 workspace 内。
- 文件写入不能允许路径穿越。
- shell 执行必须限制工作目录、超时和命令 allowlist。
- 高风险命令如 rm、curl、sudo、git 等应该被拒绝。
- 工具结果过大时应截断并记录 structured
